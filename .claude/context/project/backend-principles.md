# Backend Principles — Go + Clean Architecture\n+\n+Last updated: 2026-03-23\n+\n+## Code style\n+- Context first param: `func Do(ctx context.Context, ...)`.\n+- Error wrapping always: `fmt.Errorf(\"context: %w\", err)`.\n+- No `panic` outside `main` startup; no `init` for dependency setup.\n+- No globals; constructor injection only.\n+- Package names: lowercase, single word (`ppdb`, `selection`, `payment`).\n+\n+## Layer rules\n+- Domain imports nothing internal.\n+- Service depends on domain interfaces only (never concrete repos/adapters).\n+- Handler depends on domain interfaces only; no business logic in handlers.\n+- Repository implementations live in `internal/repository/postgres`; SQL only.\n+- Adapters implement domain interfaces for external systems (payment, WhatsApp, selection).\n+\n+## Multi-tenant & data safety\n+- Repository signatures: `schoolID` first argument; queries must start with `WHERE school_id = $1`.\n+- NIK/KK encryption (AES-256-GCM) and masking on output using `masking.MaskNIK`.\n+- `ppdb_audit_logs` is append-only; status changes log-before-update inside same transaction.\n+\n+## Testing\n+- Table-driven tests; co-locate `*_test.go` next to implementation.\n+- Integration tests use real PostgreSQL; no DB mocks.\n+- Run `go test -race ./...` in CI.\n+\n+## Observability & perf\n+- Structured logging via slog; include `school_id` in log context.\n+- Prometheus metrics and OpenTelemetry tracing where applicable.\n+- Avoid N+1 queries; add indexes for common patterns.\n*** End Patch ***!
